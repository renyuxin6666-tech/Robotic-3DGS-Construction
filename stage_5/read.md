# **四、阶段 5 的完整推理流程（一步不跳）**

**下面是** **真实执行顺序** **。**

---

## **Step 5.1 真实图像 → 轮廓（Preprocess）**

### **你在做什么**

把真实相机图像**强制转换**为一个：

* 黑色树枝
* 白色背景
* 尽量接近阶段 1 渲染风格的轮廓图

### **原则（非常重要）**

> **宁可丢细节，也不要引入新信息**

阶段 5 的轮廓应该“像合成数据”，而不是“像照片”。

---

## **Step 5.2 轮廓 → embedding（模型前向）**

```
embedding_query = Model(silhouette)
```

此时你得到的是：

* 一个和阶段 4 向量库 **同空间、同语义** 的向量

> 这是整个系统的“通用几何语言”。

---

## **Step 5.3 embedding → Top-K 检索（核心）**

### **检索行为**

```
TopK = Search(embedding_query, vector.index)
```

返回的是 K 条记录：

```
[
  (embedding_id_1, distance_1),
  (embedding_id_2, distance_2),
  ...
]
```

每一条都能反查：

```
→ branch_id
→ view_id
→ camera_pose
```

---

### **你此刻得到的不是“答案”，而是：**

> **一组最有可能的“树枝 + 姿态假设”**

---

## **Step 5.4 粗姿态估计（Coarse Pose）**

### **最简单、也是最稳健的方式**

* **统计 Top-K 中 ****branch_id 的投票**
* 选票数最多的作为识别结果
* 取其对应的 camera_pose 作为 **粗姿态**

```
branch_id* = mode(branch_id in TopK)
R_coarse   = pose of best matching template
```

> 这一步已经可以达到：

* > 树枝识别：高可靠
  >
* > 姿态误差：~5–15°
  >

---

## **Step 5.5 姿态精修（可选，但强烈推荐）**

### **这是“从可用 → 工业级”的关键一步**

#### **核心思想（你前面已提到）**

> **Render-and-Compare**

### **精修过程逻辑**

1. **使用粗姿态 **R_{coarse}
2. 在后台渲染对应树枝的轮廓
3. 与真实轮廓进行形状对齐
4. 微调旋转参数，使两者轮廓最接近

### **输出**

```
R_refined
```

可将姿态误差压到：

* 1–2°
* 局部可见仍然稳定

---

## **Step 5.6 置信度评估（Confidence）**

**你需要一个** **是否“相信这次结果”的判断** **。**

常用信号：

* Top-1 与 Top-2 的距离差
* 同一 branch_id 在 Top-K 中的占比
* 精修前后轮廓重合度提升幅度

输出一个：

```
confidence ∈ [0, 1]
```

# **六、阶段 5 的验收标准（非常实用）**

在进入阶段 6（新树枝入库）前，至少完成以下检查：

### **✅ 检查 1：局部可见性**

* 只给 30–40% 可见轮廓
* 仍能稳定返回正确 branch_id

### **✅ 检查 2：姿态一致性**

* 同一棵树枝，多次拍摄
* 输出姿态变化连续、无跳变

### **✅ 检查 3：拒识能力**

* **输入一根****未入库的树枝**
* confidence 明显下降
