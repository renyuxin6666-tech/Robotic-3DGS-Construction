# 阶段 1 渲染优化方案：自动居中与自适应相机

## 1. 问题分析
当前渲染流程存在以下问题：
- **位置偏差**：原始 `.obj` 模型的坐标原点可能不在树枝的几何中心，导致渲染时物体偏离画面中心。
- **视野截断**：不同树枝尺寸差异较大，使用固定相机距离（如 `radius: 1.5`）会导致大物体超出画面、小物体在画面中占比过小。

## 2. 解决方案：归一化 (Normalization)
为了保证每一张轮廓图都是**完整**且**居中**的，我们需要引入“归一化”步骤。我们不需要真的去改变模型的几何数据（如体素化），只需要在渲染前对场景布局进行标准化调整。

### 核心步骤

#### 第一步：几何中心归零 (Re-centering)
无论原始模型在什么位置，导入后强制将其**几何中心 (Bounding Box Center)** 移动到世界坐标原点 `(0, 0, 0)`。

#### 第二步：计算包围球半径 (Bounding Sphere)
计算物体所有顶点到原点的最大距离 $R_{max}$。这个半径定义了一个能够完全包裹物体的最小球体。

#### 第三步：自适应相机距离 (Adaptive Camera Distance)
根据相机视场角 (FOV) 和包围球半径 $R_{max}$，动态计算相机应该放置的距离 $D$。

公式如下：
$$ D = \frac{R_{max}}{\sin(\frac{FOV}{2})} \times \text{SafetyMargin} $$

其中 `SafetyMargin` (安全边距) 通常取 1.1 ~ 1.2，留出边缘呼吸空间，防止贴边。

## 3. 可行性评估
- **Blender API 支持**：Blender 提供了成熟的 `origin_set` 和包围盒计算接口，实现成本极低。
- **计算开销**：只涉及简单的顶点遍历或包围盒计算，耗时可忽略不计。
- **风险**：无。这是一种标准的计算机图形学预处理管线。

## 4. 实施计划

### 修改 `src/render/scene.py`
增加 `normalize_object(obj)` 函数：
1. 重置物体原点到几何中心。
2. 将物体位置移至 `(0,0,0)`。
3. 返回物体的包围半径 $R_{max}$。

### 修改 `scripts/render_dataset.py`
1. 在 `import_obj` 后调用 `normalize_object`。
2. 获取 $R_{max}$。
3. 计算动态距离 `cam_dist` 替代配置文件中的固定 `radius`。
