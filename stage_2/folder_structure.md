# **阶段 2 的核心设计思想（必须理解）**

## **1️⃣ “局部样本”不是新标签**

这是阶段 2 最容易犯错的地方。

* 局部裁剪 ≠ 新视角
* **局部样本 ****仍然对应同一个 3D 位姿**

> 换句话说：

> **裁掉信息，但不改变几何真相**

这正是模型学会“用不完整信息推断完整结构”的关键。

---

## **2️⃣ 局部样本的“存在意义”**

你在阶段 2 实际是在构造这样一种监督信号：

> 「这是同一根树枝、同一姿态，但你只能看到它的一部分」

后续训练中会用它来构造：

* 正样本对：
  * 全局轮廓 ↔ 局部轮廓
  * 局部 A ↔ 局部 B
* 负样本对：
  * 不同树枝（无论局部还是全局）

---

## **3️⃣ 数据集划分规则（极其重要）**

**必须按 branch_id 划分 train / val / test**

❌ 错误方式：

* 同一根树枝的不同视角分别进 train / test

✅ 正确方式：

* **一根树枝 ****只存在于一个 split 中**

原因很简单：

> 否则模型会“记住树枝”，而不是“学会泛化几何规律”。

---

# **阶段 2 输出的 index.jsonl（接口约定）**

阶段 2 最重要的文件不是图片，而是：

```
data/prepared/*/index.jsonl
```

每一行至少应包含：

```
sample_id        # 样本唯一ID
image_path       # 图像路径
branch_id        # 树枝ID
view_id          # 原始全局视角ID
is_local         # true / false
visible_ratio    # 可见比例（0~1）
pose             # 原始相机位姿（从阶段1继承）
```

> 后续 **训练、检索、姿态估计**

> 都只依赖这个 index 文件，不直接扫描目录

---

# **阶段 2 完成的“验收标准”**

在进入阶段 3 之前，你应该能做到：

1. **随机打开一条 **index.jsonl**：**
   * 能明确知道它是 **哪根树枝**
   * **是 ****哪个视角**
   * **是 ****全局还是局部**
2. 可视化对比：
   * 全局轮廓 vs 局部轮廓
   * 人眼仍然能判断“这是同一根树枝”
3. train / val / test 中 **branch_id 不重叠**

如果这三点成立，阶段 2 就是**工程上合格的**。
